# Market Opportunity - 市场机会判断

## 核心目标

**判断这个市场对谁有机会：大厂？创业公司？AI Native创业公司？**

```
┌─────────────────────────────────────────────────────────────────┐
│                    市场机会判断流程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  前置条件: 已完成市场规模测算（使用 /market-sizing）            │
│                                                                 │
│  Step 1: 增速判断 → 市场在变大还是变小？多快？                  │
│      ↓                                                          │
│  Step 2: 竞争格局 → 谁在吃蛋糕？份额在集中还是分散？            │
│      ↓                                                          │
│  Step 3: 护城河分析 → 大厂凭什么守住份额？                      │
│      ↓                                                          │
│  Step 4: 创业机会 → 还有哪些缝隙？AI能改变什么？                │
│      ↓                                                          │
│  输出: 机会判断报告                                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

# Step 1: 增速判断

## 1.1 历史增速分析

**必须收集龙头公司近5年数据：**

```
┌─────────────────────────────────────────────────────────────────┐
│ 年份    收入      YoY增速    备注                               │
│ ─────────────────────────────────────────                       │
│ 2020    $XX亿     +XX%      (疫情影响？)                        │
│ 2021    $XX亿     +XX%                                          │
│ 2022    $XX亿     +XX%                                          │
│ 2023    $XX亿     +XX%                                          │
│ 2024    $XX亿     +XX%                                          │
│                                                                 │
│ 趋势：□ 加速  □ 稳定  □ 减速  □ 负增长                         │
└─────────────────────────────────────────────────────────────────┘
```

## 1.2 增长驱动力分析

**增长来自哪里？还有多少空间？**

| 驱动力 | 当前状态 | 未来空间 |
|--------|---------|---------|
| 客户数增长 | 还有多少潜在客户未触达？ | 见顶 / 仍有空间 |
| ARPU提升 | 客户付费意愿能否提高？ | 有空间 / 已充分 |
| 渗透率提升 | 品类渗透率多少？ | 早期(<30%) / 成熟(>60%) |
| 场景扩展 | 能切入新场景吗？ | 有 / 无 |

## 1.3 增速阶段判断

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  □ 高速增长期（CAGR > 30%）                                     │
│    特征：渗透率低、客户数快速增加、龙头也在高速成长             │
│    机会：市场够大，可以正面竞争                                 │
│    代表：2015年的云计算、2020年的协作工具                       │
│                                                                 │
│  □ 稳定增长期（CAGR 15-30%）                                    │
│    特征：渗透率中等、增长来自ARPU提升和存量扩展                 │
│    机会：找细分市场、差异化定位                                 │
│    代表：当前的SaaS、电商                                       │
│                                                                 │
│  □ 成熟期（CAGR < 15%）                                         │
│    特征：渗透率高、龙头增速放缓、开始打价格战                   │
│    机会：颠覆性创新或不进入                                     │
│    代表：传统软件、PC硬件                                       │
│                                                                 │
│  □ 衰退期（负增长）                                             │
│    特征：被替代品侵蚀、客户数下降                               │
│    机会：不建议进入                                             │
│    代表：功能机、DVD租赁                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

# Step 2: 竞争格局分析

## 2.1 市场集中度

**关键指标：CR3/CR5变化趋势**

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  玩家        2020    2024    变化     含义                      │
│  ──────────────────────────────────────────                     │
│  龙头1       XX%     XX%     +Xpp     在吃掉市场                │
│  龙头2       XX%     XX%     ±Xpp     稳定/下滑                 │
│  龙头3       XX%     XX%     -Xpp     在失去份额                │
│  其他        XX%     XX%     -Xpp     长尾被挤压                │
│                                                                 │
│  CR3: XX% → XX%                                                 │
│  趋势: □ 在集中  □ 稳定  □ 在分散                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 2.2 格局判断

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  □ 一家独大（CR1 > 50%）                                        │
│    例：搜索(Google 92%)、独立站SaaS(Shopify 52%)               │
│    创业难度：★★★★★                                              │
│    策略：避开正面，找垂直/地域细分                              │
│                                                                 │
│  □ 双寡头（CR2 > 70%）                                          │
│    例：外卖(美团+饿了么)、云(AWS+Azure)、打车(滴滴+Uber)       │
│    创业难度：★★★★                                               │
│    策略：等待格局变化或政策机会                                 │
│                                                                 │
│  □ 多强并立（CR5 40-60%）                                       │
│    例：CRM、ERP、项目管理                                       │
│    创业难度：★★★                                                │
│    策略：差异化定位仍有机会                                     │
│                                                                 │
│  □ 高度分散（CR10 < 30%）                                       │
│    例：餐饮SaaS、本地服务、垂直行业软件                         │
│    创业难度：★★                                                 │
│    策略：快速抢占，但警惕巨头入场                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 2.3 份额变化方向（最关键！）

**龙头增速 vs 行业增速，决定创业窗口**

| 对比 | 含义 | 对创业公司 |
|------|------|-----------|
| 龙头增速 > 行业增速 | 份额在集中 | ❌ 窗口在关闭 |
| 龙头增速 ≈ 行业增速 | 格局稳定 | ⚠️ 需要差异化 |
| 龙头增速 < 行业增速 | 份额在分散 | ✅ 机会出现 |

# Step 3: 护城河分析

## 3.1 护城河清单

**逐项评估龙头的护城河强度：**

```
┌─────────────────────────────────────────────────────────────────┐
│ 护城河类型              │ 强度  │ 说明                          │
├─────────────────────────────────────────────────────────────────┤
│                         │       │                               │
│ 1. 网络效应             │ 强/弱 │                               │
│    用户侧：用户越多越有价值？                                   │
│    供给侧：开发者/商家生态？                                    │
│                         │       │                               │
│ 2. 规模经济             │ 强/弱 │                               │
│    边际成本：规模大→成本低→价格低？                            │
│    议价能力：对供应商有议价权？                                 │
│                         │       │                               │
│ 3. 转换成本             │ 强/弱 │                               │
│    数据迁移：数据能轻易导出吗？                                 │
│    学习成本：换产品要重新学习？                                 │
│    集成成本：和其他系统深度绑定？                               │
│                         │       │                               │
│ 4. 品牌认知             │ 强/弱 │                               │
│    品类=品牌？（"建站=Shopify"）                                │
│    信任溢价：愿意为品牌多付钱？                                 │
│                         │       │                               │
│ 5. 数据壁垒             │ 强/弱 │                               │
│    独有数据：别人拿不到的数据？                                 │
│    数据飞轮：数据让产品更好用？                                 │
│                         │       │                               │
│ 6. 牌照/合规            │ 强/弱 │                               │
│    需要特殊资质？（支付牌照等）                                 │
│    监管限制新进入者？                                           │
│                         │       │                               │
│ 7. 资本壁垒             │ 强/弱 │                               │
│    龙头累计投入多少？                                           │
│    新进入者需要多少资本追平？                                   │
│                         │       │                               │
└─────────────────────────────────────────────────────────────────┘
```

## 3.2 护城河强度评分

```
强护城河数量    │ 评估           │ 创业策略
───────────────┼────────────────┼─────────────────────────
5+ 项          │ 几乎不可攻破   │ 不要正面竞争
3-4 项         │ 正面进攻很难   │ 找侧翼：垂直/地域/客群
1-2 项         │ 有差异化空间   │ 可以尝试差异化竞争
0 项           │ 市场仍在形成   │ 快速抢占
```

# Step 4: 创业机会判断

## 4.1 机会矩阵

```
                        市场增速
                    高 (>20%)      低 (<20%)
                 ┌─────────────┬─────────────┐
    份额         │             │             │
    在分散       │  ✅ 最佳    │  ⚠️ 谨慎    │
                 │  快速抢占   │  需差异化   │
                 ├─────────────┼─────────────┤
    份额         │             │             │
    在集中       │  ⚠️ 找缝隙  │  ❌ 不进入  │
                 │  垂直切入   │  除非颠覆   │
                 └─────────────┴─────────────┘
```

## 4.2 创业机会类型

### Type A: 垂直行业机会

**大厂做不好/不想做的细分行业**

```
判断标准：
□ 行业有特殊需求（合规、流程、术语）
□ 市场规模对大厂不够有吸引力（<$10亿）
□ 需要深度行业know-how
□ 客户愿意为专业化付溢价

成功案例：
- Toast（餐饮）vs 通用POS
- Veeva（医药）vs Salesforce
- Procore（建筑）vs 通用PM
- ServiceTitan（家庭服务）vs 通用CRM
```

### Type B: 地域市场机会

**大厂渗透率低的区域**

```
判断标准：
□ 本地支付/物流/合规需求
□ 语言/文化适配需求
□ 大厂尚未重点投入
□ 本地竞争对手也不强

成功案例：
- Razorpay（印度支付）vs Stripe
- Grab（东南亚）vs Uber
- Nubank（拉美）vs 传统银行
```

### Type C: 客群细分机会

**大厂覆盖不好的客户群**

```
判断标准：
□ 特定客群需求未被满足
□ 大厂产品对该客群过度复杂/过度简单
□ 该客群有独特获客渠道

成功案例：
- Vercel（开发者）vs AWS
- Notion（小团队）vs Confluence
- Figma（设计师）vs Adobe
- Gumroad（创作者）vs Shopify
```

### Type D: 技术代际机会（AI Native）

**AI能根本性改变的环节**

```
┌─────────────────────────────────────────────────────────────────┐
│                    AI创业机会判断框架                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  问题1: AI能10x改善哪个环节？                                   │
│                                                                 │
│  □ 内容生成：文案、图片、视频、代码                             │
│  □ 决策支持：选品、定价、投放、风控                             │
│  □ 自动化：客服、运营、分析、报告                               │
│  □ 个性化：推荐、定制、匹配、搜索                               │
│                                                                 │
│  问题2: 大厂在这个环节的AI能力？                                │
│                                                                 │
│  □ 已有AI且深度集成 → ❌ 难以竞争                               │
│  □ 有AI但非核心功能 → ⚠️ 可能被追上                            │
│  □ 尚未布局AI → ✅ 窗口期机会                                   │
│                                                                 │
│  问题3: AI需要的数据从哪来？                                    │
│                                                                 │
│  □ 需要大厂独有数据 → ❌ 没法做                                 │
│  □ 可用公开数据训练 → ⚠️ 护城河弱                              │
│  □ 可从客户处积累 → ✅ 有数据飞轮可能                           │
│                                                                 │
│  问题4: AI能否改变商业模式？                                    │
│                                                                 │
│  □ 只是功能增强 → 大厂容易跟进                                  │
│  □ 改变交付方式 → 可能有颠覆机会                                │
│  □ 改变价值分配 → 最大的机会                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

# 案例：独立站SaaS市场

```
执行摘要：
┌─────────────────────────────────────────────────────────────────┐
│ 市场规模：$45亿（订阅），$170亿（全栈）                         │
│ 市场增速：26% CAGR（☑稳定增长）                                │
│ 竞争格局：☑一家独大（Shopify 52%）                             │
│ 份额趋势：☑在集中（Shopify份额从45%→52%）                      │
│ 创业机会：☑谨慎（正面竞争无机会，需找细分）                    │
│ AI机会：☑有限（Shopify已在布局AI）                             │
└─────────────────────────────────────────────────────────────────┘

结论：
- 通用独立站SaaS：❌ 不适合创业
- 垂直行业（餐饮/美业）：✅ 有机会
- 地域市场（东南亚/拉美）：✅ 有机会
- AI Native：⚠️ 谨慎（大厂已在跟进）
```
