# AI Funding Tracker - 硅谷AI融资动态追踪

**模式判断：**
- `daily` → 日报模式
- `weekly` → 周报模式（默认）
- `monthly` → 月报模式

---

## 核心定位

为VC从业者提供美国/硅谷AI一级市场融资动态，重点关注：
- **赛道分类**：AI Infra、AI Agent、AI Coding、AI Healthcare、B端应用、C端应用
- **轮次重点**：全阶段覆盖，尤其关注A/B轮（共识即将形成阶段）
- **趋势洞察**：从融资事件中提炼行业趋势信号

## 重点赛道深度分析框架

### AI Agent 赛道深度分析

**赛道演进阶段（必须判断当前处于哪个阶段）：**

```
阶段1: Copilot时代 (2022-2024)
├── 特征: 人机协作，AI辅助人类决策
├── 代表: GitHub Copilot, ChatGPT
└── 投资逻辑: 提升效率，人仍是主体

阶段2: Agent基础设施 (2024-2025) ← 当前主要阶段
├── 特征: Agent需要的"水电煤"开始建设
├── 子赛道:
│   ├── 数据获取: Nimble (实时web数据)
│   ├── 执行环境: Browser-use, Computer-use
│   ├── 记忆存储: 向量数据库, 长期记忆
│   ├── 工具调用: Function calling平台
│   └── 安全治理: Agent安全, 权限管理
└── 投资逻辑: Picks & shovels, 确定性更高

阶段3: 垂直Agent应用 (2025-2026) ← 正在进入
├── 特征: 特定场景的端到端自动化
├── 代表: Kana (营销), Glean (企业搜索)
└── 投资逻辑: 场景明确, ROI可量化

阶段4: 通用Agent (2027+) ← 尚未到来
├── 特征: 跨场景自主决策
├── 风险: 可靠性、安全性、成本
└── 投资逻辑: 长期布局, 高风险高回报
```

**本周Agent赛道需要回答的问题：**

1. **融资集中在哪一层？** (应用/平台/基础设施)
2. **有无新的子赛道出现？** (如Agent安全、Agent数据)
3. **大厂动作是什么？** (自建 vs 收购 vs 投资)
4. **可靠性问题有无突破？** (关键技术进展)
5. **商业化进展如何？** (ARR、客户数、使用量)

**Agent投资逻辑演变：**

| 时期 | 投资逻辑 | 代表案例 |

| 2025 | 投Agent Infra | Nimble, AgentOps |
| 2026 | 投垂直Agent + Infra | 分化明显 |

**风险提示（每周需评估）：**

- 安全风险: Rogue agents, Shadow AI
- 大厂挤压: OpenAI/Anthropic自建Agent能力

├── 特征: Legacy代码翻译, 可验证迁移
├── 代表: Code Metal
├── TAM: $100B+ 存量代码迁移市场
└── 投资逻辑: 企业刚需, 高客单价

├── 特征: 自然语言编程, 代码作为中间产物

| 时期 | 投资热点 | 核心逻辑 | 风险 |

**竞争格局关键变量：**

1. **Microsoft/GitHub的防守策略**

2. **模型能力是否会趋同**

   - 如果某家模型大幅领先，绑定该模型的工具受益

3. **开源IDE的威胁**
   - Zed, Void等开源项目
   - 开发者对隐私的担忧

4. **企业采购的转折点**
   - 从个人付费到企业集采
   - 合规、安全、审计需求

**风险提示（每周需评估）：**
- Copilot降价/免费风险
- 代码版权争议（训练数据来源）
- 模型能力趋同导致差异化困难
- "AI工程师"可靠性仍不足以替代人类

## 执行流程

### Step 1: 数据收集（必须执行）

**所有模式都必须依次抓取以下4个RSS源：**

```
WebFetch 1: https://techcrunch.com/feed/
Prompt: 提取所有AI相关的融资新闻，包括：公司名、融资金额、轮次、领投方、日期、一句话描述

WebFetch 2: https://news.crunchbase.com/feed/
Prompt: 提取所有融资新闻，包括：公司名、金额、轮次、投资人、估值（如有）

WebFetch 3: https://feeds.feedburner.com/venturebeat/SZYF
Prompt: 提取AI融资新闻，包括：公司名、金额、轮次、投资人

WebFetch 4: https://sifted.eu/feed
Prompt: 提取融资新闻，包括：公司名、金额、轮次、投资人（欧洲市场）
```

**根据模式筛选时间范围：**
- 日报：仅保留过去24小时的融资事件
- 周报：保留过去7天的融资事件
- 月报：保留过去30天的融资事件

### Step 2: 数据整理

对每条融资事件提取：
- 公司名称
- 融资金额
- 融资轮次
- 领投方
- 赛道分类
- 一句话亮点

**创始人背景（重要）：**

对于A/B轮及以上的重点交易，必须抓取创始人信息：
```
WebFetch: https://[公司官网]/about
WebFetch: https://www.crunchbase.com/person/[创始人名]
```

需要提取的创始人信息：
- 姓名和职位
- 之前的公司/经历（重点关注：大厂背景、连续创业者、学术背景）
- 核心能力标签（如：ex-Google、Stanford PhD、2x founder等）

**创始人背景的投资信号意义：**
| 背景类型 | 投资信号 |
|----------|----------|
| 大厂核心团队 | 有规模化经验，懂如何从0到1再到100 |
| 连续创业者 | 知道坑在哪，执行力强 |
| 学术大牛 | 技术壁垒高，适合frontier tech |
| 行业老兵 | 懂客户痛点，BD能力强 |

### Step 3: 趋势分析

**分析维度：**
1. **赛道热度**：哪些赛道融资最活跃
2. **轮次分布**：A/B轮占比，早期vs成长期
3. **资金规模**：平均融资额变化
4. **投资人动向**：哪些VC最活跃
5. **新兴信号**：首次出现的新概念/方向

**重点关注（A/B轮）：**
- 这些公司代表的是"共识即将形成"的方向
- 分析为什么投资人在这个阶段下注
- 提炼背后的行业判断

### Step 3.5: 投资逻辑反推（核心价值）

**对每个赛道必须回答：为什么VC在这个方向下注？**

抓取以下来源获取投资逻辑：
```
WebFetch: https://a16z.com/ai/
WebFetch: https://www.bvp.com/atlas
WebFetch: https://techcrunch.com/tag/[赛道关键词]/
```

**投资逻辑分析框架：**

| 维度 | 需要回答的问题 |
|------|----------------|
| **市场规模** | TAM多大？增速如何？ |
| **时机判断** | 为什么是现在？什么变化使这个方向可行？ |
| **竞争格局** | 谁是玩家？护城河在哪？ |
| **商业模式** | 如何变现？毛利结构？ |
| **风险点** | 主要风险是什么？ |

**各赛道投资逻辑参考：**

**AI Infra:**
- 逻辑：算力需求爆发 + Nvidia依赖风险 + 推理成本下降需求
- 关键指标：GPU利用率、推理成本/token、模型训练效率
- 风险：大厂自建、开源替代

**AI Agent:**
- 逻辑：从Copilot到自主执行的范式转变
- 关键指标：任务完成率、人工介入率、ROI
- 风险：可靠性不足、成本过高、安全问题

**AI Coding:**
- 逻辑：开发者市场大($600B+) + 生产力提升可量化
- 关键指标：代码接受率、开发速度提升%
- 风险：大厂免费产品（Copilot）、差异化难

**B端应用:**
- 逻辑：垂直AI毛利高(65%) + 替代人力服务($万亿市场)
- 关键指标：ARR增速、ACV、NRR
- 风险：销售周期长、行业know-how壁垒

**C端应用:**
- 逻辑：用户习惯形成 + 订阅付费意愿提升
- 关键指标：DAU、付费转化率、LTV
- 风险：获客成本高、留存难、大厂挤压

**AI Hardware:**
- 逻辑：供应链安全 + 端侧AI需求 + 专用芯片效率优势
- 关键指标：性能/瓦特、价格/性能比
- 风险：研发周期长、资本密集、Nvidia护城河

## 执行规范

### 数据准确性
- 所有融资数据必须有来源
- 金额单位统一（$M = 百万美元）
- 如数据冲突，标注多个来源

### 分析专业性
- 避免空洞描述（"非常火热"、"备受关注"）
- 用具体数字说话
- 趋势判断要有依据

### 格式规范
- 不使用emoji（保持专业）
- 使用纯文本分隔线（————）
- 适合Telegram阅读的简洁排版
- 每条消息控制在合理长度

### 时效性
- 日报：当日数据
- 周报：过去7天数据
- 月报：过去30天数据

---

## 质量标准

### 必须包含
- 具体融资事件（公司、金额、轮次、投资人）
- 赛道分类和热度
- A/B轮重点项目分析
- 趋势信号判断

### 禁止
- 编造融资数据
- 没有来源的信息
- 过于笼统的分析
- 超长的段落

### 价值导向
- 帮助VC快速了解市场动态
- 识别共识形成中的方向
- 发现潜在投资机会
